Pipeline / Architecture
Submit a YouTube URL; the API-first client checks daily quota and rate limits→Fall back to Playwright/Selenium stealth scraping when the API is constrained→Queue ingestion, analysis, and embedding jobs onto Celery + Redis→Persist channels, videos, comments, and captions via async SQLAlchemy repositories→Run sentiment analysis and topic modeling over comments and captions→Embed sources and serve RAG / VQA / chat answers through the unified LLM client→Stream task progress over WebSocket and surface results via FastAPI routes
Key Capabilities

Dual-track ingestion

API-first with Playwright/Selenium scraping fallback, rate limiting, and daily quota tracking

Async orchestration

Celery + Redis across scraping/analysis/priority queues with Beat scheduling and Flower monitoring

Clean Architecture

Layered separation of domain, services, infrastructure repositories, and API routers

Unified LLM client

Pluggable abstraction over OpenAI, LLMVendor LLMProvider, and local models for RAG, VQA, and chat

NLP analytics

Sentiment analysis and topic modeling with transformers, sentence-transformers, spaCy, and scikit-learn

Containerized stack

docker-compose deployment with Prometheus + Grafana monitoring and an Nginx reverse proxy
